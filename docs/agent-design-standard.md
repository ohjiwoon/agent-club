# Agent Club — 에이전트 설계 표준서

> 이 문서는 모든 에이전트를 설계할 때 옆에 두고 참고하는 기준서입니다.
> 새 에이전트를 만들기 전 반드시 이 문서를 먼저 읽으세요.

---

## 1. Agent Club이란?

**Agent Club**은 지운의 AI 에이전트 팀을 시각적으로 관리하는 대시보드 웹사이트입니다.

| 기능 | 설명 |
|------|------|
| 조직도 시각화 | 에이전트 팀 계층 구조를 한눈에 파악 |
| 에이전트 정보 열람 | 각 에이전트의 역할, 상태, 소속 팀 확인 |
| 결과물 보관함 | 에이전트가 생성한 파일을 대시보드에서 바로 열람 |
| 진척도 확인 | 작업 상태를 실시간으로 모니터링 (향후 확장) |

> ⚠️ Agent Club은 에이전트에게 업무를 직접 지시하는 도구가 아닙니다.
> 조직과 결과물을 **보고 관리**하는 것이 핵심 용도입니다.

## 2. 조직 구조

```
┌─────────────────────────────────┐
│           CEO (지운)             │
└────────────────┬────────────────┘
                 │
                 ▼
┌─────────────────────────────────┐
│         마케팅 팀장 에이전트        │
└──────┬──────────┬───────┬───────┘
       │          │       │
       ▼          ▼       ▼
  [블로그팀]  [뉴스레터팀] [카드뉴스팀]
  에이전트     에이전트    에이전트
  N명         N명         N명
```

### 계층 정의

| 레벨 | 역할 | 현재 구성원 |
|------|------|------------|
| Level 0 | CEO | 지운 (사람) |
| Level 1 | 팀장 | 마케팅 팀장 에이전트 |
| Level 2 | 팀원 | 블로그팀 / 뉴스레터팀 / 카드뉴스팀 에이전트 |

## 3. 에이전트 설계 체크리스트

새 에이전트를 설계할 때 아래 항목을 모두 채우세요.

```
[ ] 에이전트 이름       : ___________________________
[ ] 계층 레벨          : Level 1 (팀장) / Level 2 (팀원)
[ ] 소속 팀            : ___________________________
[ ] 보고 대상          : ___________________________  (상위 에이전트 또는 CEO)
[ ] 역할 한 줄 요약     : ___________________________

[ ] 입력값 (Input)
    - 필수: ___________________________
    - 선택: ___________________________

[ ] 출력값 (Output)
    - 형식: 마크다운 (.md) 고정
    - 파일명 규칙: [주제]_[에이전트명].md

[ ] Claude 시스템 프롬프트 작성 완료
[ ] 대시보드 send_to_dashboard() 함수 포함
[ ] 대시보드에 에이전트 등록 (POST /api/agents)
```

## 4. 대시보드 API 명세

### 에이전트 등록

```
POST /api/agents
Content-Type: application/json
```

| 필드 | 필수 | 설명 | 예시 |
|------|------|------|------|
| `name` | ✅ | 에이전트 이름 | `"뉴스레터 작성 에이전트"` |
| `team` | ✅ | 소속 팀 | `"뉴스레터팀"` |
| `level` | ✅ | 조직 계층 | `"팀장"` / `"팀원"` |
| `reportsTo` | ✅ | 보고 대상 | `"마케팅 팀장 에이전트"` |
| `description` | ☑️ | 역할 설명 | `"트렌드 분석해 뉴스레터 초안 작성"` |
| `capabilities` | ☑️ | 기능 태그 배열 | `["트렌드 분석", "초안 작성"]` |
| `icon` | ☑️ | 아이콘 이름 | `"Bot"` |
| `gradient` | ☑️ | 카드 색상 | `"from-blue-500 to-cyan-500"` |

```bash
# 테스트
curl -X POST http://localhost:3000/api/agents \
  -H "Content-Type: application/json" \
  -d '{
    "name": "뉴스레터 작성 에이전트",
    "team": "뉴스레터팀",
    "level": "팀원",
    "reportsTo": "마케팅 팀장 에이전트",
    "description": "트렌드 분석해 뉴스레터 초안 작성",
    "capabilities": ["트렌드 분석", "초안 작성"]
  }'
```

---

### 파일 전송 (에이전트 → 대시보드)

```
POST /api/files
Content-Type: application/json
```

| 필드 | 필수 | 설명 | 예시 |
|------|------|------|------|
| `name` | ✅ | 파일 제목 | `"AI 트렌드 뉴스레터"` |
| `content` | ✅ | 파일 내용 (마크다운) | `"# 제목\n\n본문..."` |
| `agentName` | ✅ | 전송한 에이전트 이름 | `"뉴스레터 작성 에이전트"` |
| `team` | ☑️ | 소속 팀 | `"뉴스레터팀"` |
| `agentId` | ☑️ | 등록된 에이전트 ID | `"agent-newsletter"` |
| `type` | ☑️ | 파일 타입 | `"md"` / `"txt"` / `"other"` |

```bash
# 테스트
curl -X POST http://localhost:3000/api/files \
  -H "Content-Type: application/json" \
  -d '{
    "name": "AI 트렌드 뉴스레터",
    "content": "# 안녕하세요\n\n첫 번째 뉴스레터입니다.",
    "agentName": "뉴스레터 작성 에이전트",
    "team": "뉴스레터팀",
    "type": "md"
  }'
```

## 5. 현재 에이전트 설계서

### [Level 1] 마케팅 팀장 에이전트

| 항목 | 내용 |
|------|------|
| **이름** | 마케팅 팀장 에이전트 |
| **계층** | Level 1 — 팀장 |
| **보고 대상** | CEO (지운) |
| **관리 팀** | 블로그팀 / 뉴스레터팀 / 카드뉴스팀 |
| **파일명** | `marketing_manager_agent.py` |
| **역할 요약** | 마케팅 콘텐츠 전반의 방향 조율 및 팀원 에이전트 산출물 취합·보고 |
| **기능 태그** | 콘텐츠 전략, 팀 조율, 결과 취합, 보고서 생성 |

**시스템 프롬프트 (예시)**
```
당신은 마케팅 팀장 에이전트입니다.
블로그팀, 뉴스레터팀, 카드뉴스팀의 산출물을 취합하고
CEO(지운)에게 마케팅 현황을 보고하는 역할을 합니다.

보고 형식:
# 마케팅 현황 보고
## 이번 주 완료 작업
## 팀별 산출물 요약
## 다음 주 계획
```

### [Level 2] 블로그팀

#### 블로그 생성 에이전트

| 항목 | 내용 |
|------|------|
| **이름** | 블로그 생성 에이전트 |
| **계층** | Level 2 — 팀원 |
| **소속 팀** | 블로그팀 |
| **보고 대상** | 마케팅 팀장 에이전트 |
| **파일명** | `blog_agent.py` |
| **입력값** | `topic` — 블로그 주제 및 키워드 |
| **출력값** | SEO 최적화 마크다운 블로그 글 |
| **기능 태그** | SEO 최적화, 목차 구성, 본문 작성, 태그 추천 |

**시스템 프롬프트 (예시)**
```
당신은 SEO 전문 블로그 작성 에이전트입니다.
주어진 주제와 키워드로 검색 상위 노출에 최적화된 블로그 글을 마크다운으로 작성합니다.

구성:
# [SEO 최적화 제목]
## 목차
## 서론 (독자 관심 유도)
## 본문 H2 섹션 3~5개
## 결론 및 요약
**추천 태그:** [태그1, 태그2, ...]

규칙:
- 3,000~5,000자
- 키워드 자연스럽게 3~5회 포함
- 각 섹션 마지막에 핵심 요약 1줄
```

| 항목 | 내용 |

```
## 마무리 및 CTA

규칙:
- 2,000자 내외
- 구어체, 친근한 톤
- 핵심 정보 위주로 요약
```

## 6. 새 에이전트 추가 절차

새로운 에이전트를 팀에 합류시킬 때 이 순서를 따르세요.

```
Step 1. 설계서 작성
   이 문서의 섹션 3 체크리스트를 채운다
   계층(Level), 소속 팀, 보고 대상을 명확히 정한다
   시스템 프롬프트를 구체적으로 작성한다

Step 2. 파이썬 파일 생성
   기존 에이전트 코드를 참고해 복사한다
   AGENT_NAME, AGENT_TEAM, SYSTEM_PROMPT, run() 수정
   send_to_dashboard() 함수 포함 여부 확인

Step 3. 대시보드 등록
   Agent Club → 에이전트 팀 → "에이전트 등록" 버튼 클릭
   이름, 팀, 레벨, 보고 대상, 설명, 기능 태그 입력

Step 4. 검증
   대시보드 조직도에서 계층 구조 확인
   파일 보관함에서 결과물 정상 등록 확인
```
